# Multi-File Verilog Support in Yosys Synthesis

## Overview
Updated Yosys integration to automatically detect and read **all** Verilog files in a Clash module's output directory, not just the primary module file.

## Problem
Previously, `YosysRunner.generateScript()` only read a single Verilog file passed via `options.verilogPath`. For complex Clash designs that generate multiple Verilog files (e.g., custom types, sub-modules, helper modules), this would cause Yosys to fail with missing module errors.

## Solution
Modified the script generation to:
1. Scan the entire directory containing the main Verilog file
2. Find all `.v` files (excluding `_shim` helper files)
3. Generate `read_verilog` commands for each file

## Implementation

### Before
```typescript
// Read single file only
let script = `# Yosys Synthesis Script
read_verilog ${options.verilogPath}
`;
```

### After
```typescript
// Find all Verilog files in the same directory as the main file
const verilogDir = path.dirname(options.verilogPath);
const allFiles = await fs.readdir(verilogDir);
const verilogFiles = allFiles
    .filter(f => f.endsWith('.v') && !f.includes('_shim'))
    .map(f => path.join(verilogDir, f));

// Build Yosys script
let script = `# Yosys Synthesis Script
# Generated by clash-vscode-yosys extension

# Read design files
`;

// Read all Verilog files
for (const vFile of verilogFiles) {
    script += `read_verilog ${vFile}\n`;
}
```

## File Filtering
The implementation excludes:
- **C++ shim files**: `*_shim.cpp` - Clash-generated simulation helpers
- **Non-Verilog files**: `clash-manifest.json` - Clash metadata
- Automatically includes: All `.v` files in the module directory

## Example Directory Structure
```
.clash-synth/verilog/MyModule.topEntity/
├── my_module.v              ← Main module (read)
├── my_module_types.v        ← Custom type definitions (read)
├── helper_module.v          ← Helper module (read)
├── my_module_shim.cpp       ← Simulation helper (skipped)
└── clash-manifest.json      ← Metadata (skipped)
```

## Generated Yosys Script
```tcl
# Yosys Synthesis Script
# Generated by clash-vscode-yosys extension

# Read design files
read_verilog .clash-synth/verilog/MyModule.topEntity/my_module.v
read_verilog .clash-synth/verilog/MyModule.topEntity/my_module_types.v
read_verilog .clash-synth/verilog/MyModule.topEntity/helper_module.v

# Elaborate design
hierarchy -check -top my_module
...
```

## Benefits
1. **Handles complex designs**: Supports Clash modules with multiple Verilog outputs
2. **Automatic discovery**: No manual file specification needed
3. **Backwards compatible**: Simple single-file modules still work
4. **Future-proof**: Scales with Clash's code generation

## Testing
Current test modules generate single files, but the implementation is ready for:
- Custom ADT type definitions in separate Verilog files
- Multi-module designs with component hierarchies
- Block RAM instances with separate definition files
