# Phase 4: Code Generation - Complete!

## ✅ Implementation Status: COMPLETE

Phase 4 has been fully implemented and tested. The extension can now generate Clash wrapper modules for monomorphic Haskell functions.

## Features Implemented

### 1. CodeGenerator Class
Located in [src/code-generator.ts](../src/code-generator.ts)

**Capabilities:**
- Generate complete Haskell modules with proper imports
- Create `topEntity` wrappers pointing to original functions
- Add `Synthesize` annotations with port naming
- Add `NOINLINE` and `OPAQUE` pragmas
- Smart port name generation (CLK/RST/EN for special types)
- Convert CamelCase to snake_case for Verilog compatibility
- Directory management (creates `.clash-synth/` automatically)
- File cleanup utilities

### 2. Smart Port Name Generation

The code generator intelligently names ports based on their types:

| Type Pattern | Port Name |
|--------------|-----------|
| `Clock ...` | CLK |
| `Reset ...` | RST |
| `Enable ...` | EN |
| Generic inputs | INA, INB, INC, ... |
| Output | OUT |

### 3. Module Naming

- **Module Name**: `ClashSynth_FunctionName` (PascalCase)
- **Synthesis Name**: `function_name` (snake_case for Verilog)

Example:
- Function: `plusSigned`
- Module: `ClashSynth_PlusSigned`
- Verilog entity: `plus_signed`

### 4. Generated File Structure

```haskell
{-# OPTIONS_GHC -Wno-orphans #-}

module ClashSynth_PlusSigned where

import Clash.Prelude
import qualified Example.Project

-- | Wrapper for synthesizing Example.Project.plusSigned
-- This module was auto-generated by clash-vscode-yosys extension
-- Original type: Signed 8 -> Signed 8 -> Signed 8

topEntity :: Signed 8 -> Signed 8 -> Signed 8
topEntity = Example.Project.plusSigned

{-# ANN topEntity
  (Synthesize
    { t_name = "plus_signed"
    , t_inputs = [ PortName "INA"
                 , PortName "INB"
                 ]
    , t_output = PortName "OUT"
    }) #-}

-- Make sure GHC does not apply optimizations
{-# NOINLINE topEntity #-}
{-# OPAQUE topEntity #-}
```

## User Workflow

1. **Detect Functions**: Run `Clash: Detect Functions`
2. **Select Function**: Choose a monomorphic function from the list
3. **Generate Wrapper**: Extension creates wrapper in `.clash-synth/`
4. **Options**:
   - Open the generated file
   - Proceed to Clash compilation (Phase 5)
   - Cancel

## Configuration

The extension respects workspace settings:

```json
{
  "clash-vscode-yosys.autoCleanup": false,  // Keep generated files?
  "clash-vscode-yosys.outputFormat": "verilog"  // HDL format (for Phase 5)
}
```
